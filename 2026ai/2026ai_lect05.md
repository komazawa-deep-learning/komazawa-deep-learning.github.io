y_sex = np.zeros_like(y)          # 男性 = 0

## クラス不均衡の問題

**データの内訳**

```
男性（0）：370枚 ─────────────────────── 92.5%
女性（1）： 30枚 ──                        7.5%
```

**ナイーブ分類器の罠**
> 「全員男性と予測する」だけで accuracy = **92.5%**

**→ accuracy は信頼できない指標**

**本日使う評価指標**

| 指標 | 意味 | なぜ重要か |
|------|------|---------|
| Recall（女性） | 女性を見逃さない率 | 見逃しのコストが高い |
| F1（女性） | Precision と Recall の調和平均 | 不均衡データの標準指標 |
| ROC-AUC | 閾値に依存しない総合性能 | 比較に適切 |

## PCA 前処理（全手法共通）

**なぜ PCA を使うか**

- 4,096 次元 → **100 次元**（計算速度と汎化性能を改善）
- `whiten=True`：各主成分の分散を 1 に正規化（SVM との相性）
- **全手法で同じ PCA を使う**（公平な比較のため）

**固有顔（Eigenfaces）**

```python
pca = PCA(n_components=100, whiten=True, random_state=42)
X_train = pca.fit_transform(X_train_raw)  # 訓練データで fit
X_test  = pca.transform(X_test_raw)       # テストは transform のみ
```

各主成分 = **顔の変動パターン**（固有顔）
→ 上位 100 成分で元データの約 80～90% の分散を説明

**層化分割**（`stratify=y_sex` 必須）
→ 女性30枚を訓練・テストに均等配分

## k-NN：直観

**最近傍法（k-Nearest Neighbors）**

```
テストデータが来たら：
1. 訓練データとの距離を計算（ユークリッド距離）
2. 最も近い k 個を選ぶ
3. 多数決でクラスを決定
```

**k の役割**

```
k=1：最近傍1点の多数決 → 過学習しやすい（境界が複雑）
k=大：多くの点の多数決 → 多数派（男性）に引きずられる
```

**クラス不均衡との関係**

- k 近傍の多くが男性（370枚）
- → k が大きいほど女性の Recall が下がる
- 最良 k は ROC-AUC または女性 F1 で選ぶ

**実装**：標準化不要（PCA で分散が揃っている）

---

## k-NN：k と決定境界

**k を変えたときの女性 Recall と ROC-AUC**

```
k=1   → 過学習：訓練で完璧，テストで不安定
k=5   → バランスが良い（多くの場合）
k=15+ → 多数派に引きずられ女性 Recall が低下
```

**2 次元 PCA 空間での決定境界**

- 第 1・第 2 主成分の 2 次元空間にプロット
- k-NN の境界は**ボロノイ図的な境界**（直線ではない）
- 女性クラスが小さな「島」として点在

**k-NN の特徴**

✅ 実装が簡単，直観的  
⚠️ 予測時に全訓練データと比較 → データが増えると遅い  
⚠️ 高次元生データでは距離が意味を失う（次元の呪い）  
→ PCA との組み合わせが必須

---

## SVM：マージン最大化

**Support Vector Machine**

**ロジスティック回帰との違い**

```
ロジスティック回帰：「境界の正しい側にいれば良い」
SVM：          「境界からできるだけ遠くに離れたい」
```

**マージン（余裕）を最大化する超平面**

```
      ×  ×                ○  ○
        ×   ← マージン →   ○
          ×               ○
          ↑サポートベクター（境界に最も近い点）
```

- マージンが大きい → 汎化性能が高い
- **決定に関与するのはサポートベクターのみ**
- 残りのサンプルを削除してもモデルは変わらない

---

## SVM：カーネルと不均衡対処

**RBF カーネル（ガウシアンカーネル）**

```
線形分離できない場合：高次元空間に写像して線形分離
「カーネル」：写像を内積として効率的に計算
```

**ハイパーパラメータ**

| パラメータ | 意味 | 影響 |
|-----------|------|-----|
| C | マージン違反へのペナルティ | 大 → 硬いマージン |
| γ | RBF の「広がり」 | 大 → 局所的な境界 |

→ GridSearchCV で最適値を探す

**クラス不均衡への対処**

```python
SVC(kernel='rbf',
    class_weight='balanced',   # ← 女性サンプルの重みを増加
    probability=True)
```

→ 顔認識（高次元・少数サンプル）に**最も相性が良い手法**

---

## 決定木：ジニ係数と分割基準

**「if-then ルールの木構造」で分類**

```
第1主成分 > 0.3 ?
├── Yes → 第2主成分 > -0.1 ?
│         ├── Yes → 男性
│         └── No  → 女性
└── No  → 男性
```

**分割基準：ジニ係数（Gini Impurity）**

$$\text{Gini} = 1 - \sum_k p_k^2$$

- $p_k$ = クラス $k$ の割合
- 純粋（1クラスのみ）→ Gini = 0
- 完全混合（2クラス均等）→ Gini = 0.5
- **ジニ係数が最も減少する特徴量・閾値で分割**

本データでは「PC$X$ の値 > 閾値」で分割 → 固有顔成分による判断

---

## 決定木：深さと過学習

**深さを変えたときの訓練・テスト F1（女性）**

```
depth=1 → 訓練F1:低い  テストF1:低い  → 未学習
depth=3 → 訓練F1:中    テストF1:高い  → 適切
depth=10→ 訓練F1:高い  テストF1:低い  → 過学習
深さ制限なし → 訓練F1:1.0  テストF1:大きく低下
```

**決定木の特徴**

✅ `plot_tree` で可視化・解釈できる  
✅ スケール不変（標準化不要）  
⚠️ **少数クラスの少数サンプルを丸暗記する**  
⚠️ 深くなるほど過学習が激しい  
→ **Random Forest がこの問題を解決する**

`class_weight='balanced'`：少数クラス（女性）に大きな重みを設定

---

## Random Forest：バギングの直観

**「多数の決定木の多数決」**

**バギング（Bagging = Bootstrap Aggregating）の手順**

```
訓練データ（300枚）
   ↓ 復元抽出（ランダムサンプリング）
[木1用データ] [木2用データ] ··· [木300用データ]
   ↓各自で学習（各分割でも特徴量をランダムに選ぶ）
  木1          木2              木300
   ↓多数決
  最終予測
```

**なぜ過学習しにくいか**

- 各木は**異なるサンプル・異なる特徴量**で学習
- 木同士の相関が低い → 多様な判断 → 平均で安定
- 1本の木のエラーは他の木でキャンセルされる

**決定木との違い**：並列・独立学習（バギング）

---

## Random Forest：特徴量重要度ヒートマップ

**`feature_importances_`：各主成分が分割にどれだけ貢献したか**

**重要度マップの作成**

```python
# PCA主成分の重要度を元の画像空間に逆変換
importance_face = np.dot(importances, pca.components_)  # (4096,)
importance_map  = importance_face.reshape(64, 64)
```

**可視化の解釈**

```
明るいピクセル → 男女識別に重要な顔の領域
↓
輪郭・眉・目などの領域が浮かび上がることが多い
```

**回帰の係数との違い**

| 比較軸 | 回帰の係数（線形） | RF の重要度（非線形） |
|--------|-----------------|-------------------|
| 捉える寄与 | 線形な影響のみ | 非線形な組み合わせも |
| 解釈 | $w_j$ の符号と大きさ | 分割への貢献量 |

→ 第4回で学んだ「係数の解釈」とRFの重要度を比較できる

---

## AdaBoost：ブースティングの直観

**「誤ったサンプルに重みを集中して繰り返し学習する」**

**Random Forest（バギング）との対比**

| 比較軸 | RF（バギング） | AdaBoost（ブースティング） |
|--------|-------------|------------------------|
| 学習の順序 | **並列**（独立） | **逐次**（依存） |
| 各木のデータ | ランダムサンプリング | 前の誤りに応じた重み付き |
| 組み合わせ | 等しい重みで多数決 | 精度に応じた重み付き投票 |
| 過学習への強さ | 強い | やや弱い |

**心理学的アナロジー（第4回との接続）**

```
第4回：勾配降下法 → 損失関数の勾配方向にパラメータを更新
第5回：AdaBoost  → 誤分類サンプルの重みを増やして次の木が注目
```

→ どちらも「誤りを減らす方向に学習を調整する」

---

## AdaBoost：学習曲線と弱学習器

**弱学習器（深さ1の決定木 = 決定スタンプ）**

- 1 つの特徴量・1 つの閾値だけで分類
- それ自体は「ランダムより少し良い」程度
- 200 個重ねると強い分類器になる

**クラス不均衡への対処**

```python
# class_weight パラメータがないため sample_weight で代替
sample_weight = np.where(y_train==1, n_male/n_female, 1.0)
ada.fit(X_train, y_train, sample_weight=sample_weight)
```

**staged_predict_proba：学習曲線の観察**

```
AUC
↑
│       ▲ RF の AUC（参考線）
│  ／￣￣
│ ／
│/
└──────────────────→ 弱学習器の数
```

弱学習器を追加するごとに AUC が上昇 → 最終的に RF と比較

## 5手法の総合比較

**ROC-AUC と女性 F1 での比較**

| 手法 | 特徴 | 期待される AUC |
|------|------|--------------|
| k-NN | 距離ベース | 中程度 |
| SVM | マージン最大化 | **最高水準** |
| 決定木 | ルールベース | 低め |
| Random Forest | バギング | 高い |
| AdaBoost | ブースティング | 中〜高 |

**accuracy だけ見ると全手法が「高い」→ 無意味**  
**女性 F1・ROC-AUC で差が明確に現れる**

**ROC 曲線の重ね合わせ**

- SVM が最も左上に位置する（顔認識の定番）
- RF が SVM に次ぐ
- 決定木は単独では不安定

## いつ何を使うか

**データの特性による使い分け**

| 条件 | 推奨手法 |
|------|---------|
| 高次元・少数サンプル | **SVM**（カーネル） |
| 大量データ，特徴量重要度が欲しい | **Random Forest** |
| 解釈性・可視化が必要 | **決定木**（浅く） |
| シンプルさを優先 | **k-NN** |
| ブースティングを試したい | **AdaBoost**（クリーンなデータ） |

**本日のデータ（オリベッティ男女分類）では**

1. **SVM**（PCA + RBF）：最も高性能
2. **RF**：特徴量重要度の可視化が面白い
3. 決定木・AdaBoost：単独では限界あり

**クラス不均衡がある場合は必ず**
→ `class_weight='balanced'` / `sample_weight` / `stratify=y_sex`

## まとめ

**5手法の位置づけ**

```
距離ベース：k-NN（最近傍）
マージンベース：SVM（マージン最大化 + カーネル）
木ベース：決定木 → RF（バギング）
                → AdaBoost（ブースティング）
```

**本日の重要な概念**

1. **クラス不均衡**：accuracy では評価できない → F1，ROC-AUC
2. **PCA 前処理**：高次元データに対して有効（固有顔）
3. **バギング vs ブースティング**：並列 vs 逐次
4. **特徴量重要度**：RF のヒートマップで「どこで判断しているか」を可視化
